Throughput

EvalApp

A self-tuning pipeline engine for .NET. You declare what work to do; it discovers how fast your hardware can run it and converges on the optimal concurrency at runtime, then re-adapts when load or hardware changes. No more hand-guessing thread counts that go stale the moment the environment shifts.

Zero external dependencies, in-process, one DLL. The type-state API makes whole classes of concurrency and saga bugs impossible to even write, and the pipeline resolves once into a pre-built delegate chain, so each run carries no per-call reflection or re-assembly.

30–40% more throughput than hand-tuned baselines · two tuners, hill-climbing and Thompson-sampling, that learn each machine's best concurrency and warm-start after a restart

A live, self-tuning engine, published on NuGet. Product page, quickstart, and install →

Real-time

NavPathfinder

Multi-agent pathfinding, crowd avoidance, influence maps, and combat, in one headless SDK over a single baked navmesh, replacing the four or five systems a game or simulation usually stitches together. Fully async, engine-agnostic, and built headless-first so it runs on servers and in CI with no renderer attached.

It self-tunes to a target frame budget, so there is no "max agents" constant to guess. It holds the frame and reports how much headroom is left.

~680 agents pathfinding and fighting inside an ~80 fps frame budget, under full load

Systems research

Prism

Our deepest research, and a study in taking a hard problem all the way down. Prism is a language model that trains on plain CPUs with no GPU, across a colony of ordinary machines where every node holds the whole model and switching one off leaves the rest running. The entire thing is built in C# from first principles, with the backward pass hand-derived and gradient-checked.

The hard part is a genuine distributed-systems property: gradients that merge to the bit. A slow laptop and a fast desktop contribute mathematically identical results, so a mix of mismatched machines trains one coherent model, something datacenter stacks cannot do without matched hardware. And because numbers are stored as decodable patterns, the arithmetic can be read straight out of the network rather than hidden in an opaque vector.

~3× the unseen-task generalisation of a same-size conventional transformer · every result reproducible from a committed script in seconds

Empirical research at small scale, published with fair baselines and its own limitations stated, including an experiment that overturned one of our own hypotheses. It is here as evidence of engineering depth.

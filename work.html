EvalApp · throughput

A self-tuning pipeline engine for .NET. You declare what work to do; it discovers how fast your hardware can run it and converges on the optimal concurrency at runtime, then re-adapts when load or hardware changes. No more hand-guessing thread counts that go stale the moment the environment shifts.

Zero external dependencies, in-process, one DLL. The type-state API makes whole classes of concurrency and saga bugs impossible to even write, and the pipeline exports itself as an always-correct architecture diagram.

30–40% more throughput than hand-tuned baselines · independent instances converge on a shared-resource optimum with no communication

Validated in integration and simulation testing; not yet deployed to a production client (there isn't one yet — that's the honest state).

NavPathfinder · real-time

Multi-agent pathfinding, crowd avoidance, influence maps, and combat, in one headless SDK over a single baked navmesh, replacing the four or five systems a game or simulation usually stitches together. Fully async, engine-agnostic, and built headless-first so it runs on servers and in CI with no renderer attached.

It self-tunes to a target frame budget, so there is no "max agents" constant to guess — it holds the frame and tells you how much headroom is left.

~680 agents pathfinding and fighting inside an ~80 fps frame budget, under full load

Measured in a headless siege harness. "Thousands" is plausible for pure navigation, but the honest, full-combat number is the one above.

Prism · to the metal

My deepest research, and the clearest window into how far I take this. Prism is a language model that runs on a CPU with no GPU, and trains itself across a colony of ordinary machines — every node holds the whole model, and switching one off leaves the rest running. Built entirely from scratch in pure C# with no PyTorch, no GPU, and no autograd; the whole backward pass is hand-derived and gradient-checked.

The hard part is a genuine distributed-systems property: gradients that merge to the bit. A slow laptop and a fast desktop contribute mathematically identical results, so a mix of mismatched machines trains one coherent model — something datacenter stacks cannot do without matched hardware. And because numbers are stored as decodable phasor patterns, you can read the arithmetic straight out of the network instead of staring at an opaque vector.

~3× the unseen-task generalisation of a same-size conventional transformer · every number reproducible from a committed script in seconds

Empirical research at toy scale, framed honestly: I even report an experiment that overturned one of my own hypotheses. It is here as proof of engineering depth, not as a product.

If I can make a neural network train to the exact bit across a swarm of mismatched laptops, I can make your API faster.
